# gitflow를 활용하여 webRTC git 협업

1.정의

a) GitFlow란 무엇인가?

	- GitFlow란
	
	- 데이터를 어떤 방식으로 저장하고 정렬하느냐에 따라 추출 방식 등 데이터를 처리
	및 조작하는데 필요한 코드가 달라진다.

2.자료 구조

a)  GitFlow를 하기 위한 branch 종류

	- 1.master
    2.develop
    3.feature
    4.release
    5.hotfix

b)  GitFlow 활용 방법
	

	-gitflow

	-FIFO 방식으로 인해 데이터 삭제시, 재정렬이 필요 없다.

	-운영체제(OS)에서 프로세스 스케줄링 방식을 구현하기 위해 사용된다.
